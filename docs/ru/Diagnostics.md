# Диагностика — wg-split-doctor

← [Назад на главную](Home.md)

`wg-split-doctor` отвечает на вопрос «что сломано и что чинить?» без ручного
лазания по SSH. Это **пассивный** инструмент: он только читает состояние и
**никогда не зондирует** туннели (зондирование меняет маршрутизацию и должно жить
только в демоне failover).

```sh
wg-split-doctor          # человекочитаемый отчёт (для SSH)
wg-split-doctor --json   # машинный формат (его показывает панель LuCI)
```

Код выхода = худшая серьёзность, поэтому работает и как пост-инсталляционная
проверка:

| Код | Серьёзность | Значение |
|:---:|-------------|----------|
| 0 | **OK** | работает как надо |
| 1 | **WARN** | неоптимально/устарело, маршрутизацию не ломает |
| 2 | **FIXABLE** | сломано, но самовосстановится на след. тике или одной командой |
| 3 | **FAIL** | сломано, нужно решение оператора (почти всегда — firewall) |

Каждая находка несёт `severity`, `category`, `message` и `→ fix` (что сделать).

## Что проверяется

- **config:** есть ли туннели; задана/определена ли LAN-подсеть.
- **endpoint:** интерфейс существует / поднят; возраст рукопожатия (пассивно:
  свежее = `ok`, старое = `idle`, без активной пробы); `route_allowed_ips=0` на
  пирах.
- **firewall:** туннель в зоне; на зоне `masq=1`; есть форвардинг `lan → зона`.
- **routing/killswitch:** правильные `ip rule` (анти-loop prio 1000, wg-марка
  prio 999); таблица 200 содержит ожидаемый default или blackhole; состояние
  соответствует конфигу.
- **lists:** наличие/количество/возраст ipsum, ru/cn, nozapret относительно
  минимумов и режима.
- **zapret:** доступен и работает, когда включён.
- **dnsmasq:** дроп-ин nftset присутствует в живом conf-dir, если настроены
  домены.

## Здоровая установка `blocklist`

`overall = OK`, активный путь `vpn:<iface>`, каждый туннель со свежим
рукопожатием + здоровьем `ok` + зоной firewall с masq и форвардингом LAN, сет
`ipsum` выше минимума. Для `full`/`split` логика та же — отличаются лишь режим и
то, какие списки важны.

## Полезные приёмы

- Самотест чистой логики **вне роутера** (без зависимостей):
  ```sh
  WG_SPLIT_SELFTEST=1 wg-split-doctor
  ```
- Машинный вывод для скриптов/CI:
  ```sh
  wg-split-doctor --json | jq .overall
  ```

Панель LuCI показывает этот же JSON в виде hero-баннера, цепочки, таблиц и блока
диагностики с прямыми ссылками на firewall/интерфейсы. См.
[Настройка и панель LuCI](Configuration.md), а конкретные исправления —
[Устранение неполадок](Troubleshooting.md).
