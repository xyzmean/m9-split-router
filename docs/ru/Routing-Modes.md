# Режимы маршрутизации

← [Назад на главную](Home.md)

Режим задаётся параметром `mode` и определяет, какой трафик LAN попадает в
туннель. Под капотом всё сводится к маркировке nftables в prerouting: пакет,
получивший VPN-марку (`0x40000`), уходит по `ip rule` в таблицу 200 → активный
туннель; пакет с анти-loop-маркой (`0x10000`) идёт в `main` (напрямую).

## `blocklist` (по умолчанию)

Через VPN едет **только** то, что в VPN-сетах:

- сет `ipsum` (скачанный список заблокированных/зарубежных IP);
- ваши ручные `vpn_cidr` и VPN-домены.

Всё остальное уходит в WAN, где его обрабатывает **zapret** (обход DPI). RU/CN по
определению остаются напрямую (они не в VPN-сетах). Это «умный» режим: VPN-канал
нагружается только реально заблокированными ресурсами.

```
LAN ─┬─ IP в ipsum / vpn_cidr / vpn-домен ─▶ VPN
     └─ всё остальное ─▶ WAN (+ zapret) ─▶ RU/CN и clearnet напрямую
```

## `full`

Маршрут по умолчанию — через **VPN**: вся неотмеченная LAN-трафик получает
VPN-марку. Список **direct** — это исключения, которые насильно остаются
напрямую:

- ваши `direct_cidr` и direct-домены;
- скачанный список `ru/cn` (в этом режиме он подключается как явное direct-
  исключение, иначе уехал бы в туннель).

```
LAN ─┬─ direct_cidr / ru-cn / direct-домен ─▶ WAN напрямую
     └─ всё остальное ─▶ VPN
```

Используйте, если хотите «всё через VPN, кроме отечественного».

## `split`

Через VPN едут **только** явно перечисленные VPN-списки/домены (`vpn_cidr`,
VPN-домены, и `ipsum` если он включён в этом сценарии). Никакой «политики по
умолчанию» нет — это самый избирательный режим.

```
LAN ─┬─ только явные VPN-списки/домены ─▶ VPN
     └─ всё остальное ─▶ WAN напрямую
```

## Какие сеты в каком режиме «живые»

| Сет | blocklist | full | split |
|-----|:---------:|:----:|:-----:|
| `ipsum` (VPN) | ✔ подключён | определён, не подключён | по списку |
| `ru/cn` (direct) | direct по умолчанию | ✔ явное исключение | direct по умолчанию |
| `vpn_cidr` / VPN-домены | ✔ | ✔ | ✔ |
| `direct_cidr` / direct-домены | ✔ | ✔ | ✔ |

> Поэтому, например, пустой `ipsum` в режиме `full`/`split` **не** ломает
> маршрутизацию — `doctor` это учитывает и не помечает коробку как «деградирует».
