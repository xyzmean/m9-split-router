# Настройка и панель LuCI

← [Назад на главную](Home.md)

## Предусловие: создайте туннель

wg-split **не управляет ключами и пирами** WireGuard/AmneziaWG. Сначала создайте
туннельный интерфейс обычным образом в **Network → Interfaces** (протокол
`wireguard` или `amneziawg`). Затем добавьте его в wg-split.

## Где настраивать

**Services → wg-split**. Конфигурация хранится в UCI: `/etc/config/wg-split`.
Страница состоит из живого **статус-дашборда** сверху и формы настроек снизу.

## Статус-дашборд (редизайн)

Новая панель показывает состояние «как есть» и подсказывает следующее действие:

- **Hero-баннер** — крупный итог здоровья (OK / WARN / FIXABLE / FAIL), активный
  путь человекочитаемо (`VPN активен через wg0`, `Нет туннеля — WAN с zapret`,
  `Аварийная блокировка` и т.д.), режим, состояние kill switch, счётчик сбоев,
  LAN-подсеть.
- **Визуализация цепочки** — `LAN → #1 wg0 → #2 wg1 → zapret → WAN`, где активный
  узел подсвечен зелёным, недоступный туннель — красным, под каждым туннелем
  возраст рукопожатия.
- **Таблица туннелей** — приоритет, рукопожатие, здоровье, зона firewall, masq,
  форвардинг LAN.
- **Таблица списков** — ipsum / ru-cn / nozapret: количество записей, минимум,
  возраст, статус.
- **Диагностика** — только проблемы (severity + сообщение + `→ исправление`).
  Для firewall-проблем есть кнопка **«Настройки межсетевого экрана»**, для
  отсутствующего интерфейса — **«Сетевые интерфейсы»** (прямые ссылки).
- **Авто-обновление** — панель сама опрашивает `wg-split-doctor` каждые ~8 секунд;
  переключатель **«Онлайн / Пауза»** справа в панели действий.
- **Подсказки первого запуска** — если туннелей ещё нет, панель объясняет три
  шага (создать интерфейс → добавить с приоритетом → зона firewall + форвардинг).

### Кнопки действий
`Обновить сейчас`, `Применить` (`wg-split-apply`), `Перезапустить службу`,
`Обновить ipsum/ru-cn/домены`, `Аварийно отключить` (`wg-split-disable`).

> Тема **Argon** поддерживается: панель использует CSS-переменные темы
> (границы/поверхности) с запасными значениями, поэтому одинаково корректно
> выглядит и на светлом Bootstrap, и на тёмном Argon. Форма настроек собрана в
> один аккуратный набор вкладок (Общие / Списки / Ручные записи).

## Форма настроек

### Вкладка «Общие»
| Параметр | UCI | Описание |
|----------|-----|----------|
| Режим маршрутизации | `mode` | `blocklist` / `full` / `split` — см. [Режимы](Routing-Modes.md) |
| Интервал отказоустойчивости | `interval` | секунды между тиками failover (по умолч. 180) |
| Kill switch | `killswitch` | при всех мёртвых туннелях — блокировать, а не утекать в WAN |
| Интерфейс LAN | `lan_iface` | по умолчанию `br-lan` |
| Подсеть LAN (CIDR) | `lan_cidr` | объединяется с авто-определённой подсетью моста |
| Цели health-ping | `health_target` | пингуются через каждый туннель |
| URL проверки WAN | `health_url` | curl через WAN для выбора zapret vs обычный WAN |

> **Авто-определение LAN.** wg-split объединяет вашу `lan_cidr` с реальными
> connected-маршрутами `lan_iface`. Это закрывает гочу «после смены нумерации LAN
> ничего не маршрутизируется».

### Вкладка «Списки»
Включение/URL для `ipsum` (VPN) и `ru/cn` (direct), URL списков доменов
(VPN/ignore), флаг zapret. Подробно — [Списки и zapret](Lists-and-zapret.md).

### Вкладка «Ручные записи»
`vpn_cidr`, `direct_cidr` (CIDR прямо в nft-сеты), `vpn_domain`, `direct_domain`
(в директивы nftset для dnsmasq).

### «Туннели отказоустойчивости»
Одна строка на туннель: интерфейс (из созданных в Network → Interfaces) и
приоритет. **Меньший номер приоритета побеждает.**

### «Переопределения по устройствам»
Закрепить LAN-хост по IP за `vpn` или `direct` (маркируется раньше остальных
цепочек).

## После «Сохранить и применить»

LuCI вызывает `wg-split-apply`: регенерирует `/etc/nftables.d/30-wg-split.nft` и
dnsmasq-дроп-ин из UCI, перезагружает fw4 + dnsmasq, перезапускает службу (демон
читает UCI один раз при старте) и при смене URL списков — дочитывает их в фоне.
