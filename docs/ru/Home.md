# wg-split — Вики

**wg-split** превращает роутер OpenWrt в локальный split-tunnel шлюз: трафик из
LAN политиками маршрутизируется через один из нескольких туннелей
WireGuard/AmneziaWG с **приоритетным failover**. Заблокированные/зарубежные IP
идут через VPN, RU/CN — напрямую, а опциональный **zapret** обходит DPI на WAN.
Всё настраивается локально через **UCI / LuCI** — без облака, API и токенов.

```
[устройство LAN] ─▶ [туннель #1] ─▶ … failover … ─▶ [туннель #N]
                 └▶ напрямую (WAN) ─▶ zapret-обход DPI для RU/clearnet
```

## Что нового в 2.0.0

- **Дашборд отделён от настроек** — две вкладки: **Статус** (диагностика, живой
  rx/tx по туннелям, таймлайн отказоустойчивости) и **Настройки** (форма UCI).
- **Firewall в один клик** — кнопка **«Починить автоматически»** на каждом
  замечании + CLI `wg-split-firewall {check|fix} <iface>`.
- **Журнал событий failover** — `wg-split-doctor --events`, кольцевой буфер в RAM.
- **Least-privilege ubus** — LuCI ходит в rpcd-объект `wg-split`
  (`status`/`events`/`action`) вместо широкого `file:exec`.
- **Site-to-site** — поле «VPN-подсети» для LAN пиров + проверка, что подсеть
  реально маршрутизируется в туннель.
- **Транспорт-агностичный шов** — у эндпоинта поле `type` (по умолч. `wg`),
  диспетчер `ep_*` готовит почву под бэкенд sing-box в 3.0.0.

Конфиги 1.7.x продолжают работать без изменений — схема UCI росла только
аддитивно. Полный список — в `CHANGELOG.md` репозитория.

## Как это устроено в двух словах

Маркировка nftables → `ip rule` → таблица 200 → активный туннель. Политические
цепочки регенерируются из UCI скриптом `wg-split-apply`; служба `wg-split` крутит
цикл failover и самовосстанавливает IP-сеты.

- **`blocklist` (по умолчанию):** через VPN едет только сет `ipsum`
  (заблокированные/зарубежные IP); всё остальное уходит в WAN с zapret. RU/CN —
  напрямую.
- **`full`:** маршрут по умолчанию через VPN, список direct — исключения.
- **`split`:** через VPN едут только ваши явные VPN-списки/домены.

## Важная оговорка про firewall

wg-split управляет **маршрутизацией** (марки + таблица 200), но **никогда не
трогает межсетевой экран**. Туннельный интерфейс всё равно должен быть в зоне
firewall (с `masq=1`) и иметь форвардинг `lan → эта зона`, иначе fw4 отбросит
форвардинг LAN→туннель, и туннель будет выглядеть «мёртвым», хотя он поднят. Это
**главная причина** проблем после перепрошивки — см.
[Устранение неполадок](Troubleshooting.md).

> Проект принципиально остаётся на **shell + LuCI** (а не Rust/Vue): оба пакета
> собираются как `PKGARCH:=all` — один билд = один `.apk` на все архитектуры,
> минимальный размер на flash, всё читается и правится прямо на роутере. Это
> осознанное архитектурное решение под платформу OpenWrt.
