# Отказоустойчивость

← [Назад на главную](Home.md)

Служба `wg-split` (procd-демон `wg-split-failover --daemon`) каждый тик
(`interval`, по умолч. 180 с) выбирает лучший здоровый туннель и перестраивает
маршруты. Туннели берутся из секций `endpoint` в UCI, **меньший приоритет
побеждает**.

## Лестница состояний (на каждом тике)

1. **Текущий туннель здоров** → оставить его.
2. **Текущий завис** → СНАЧАЛА перезапустить его (`ifdown`/`ifup` — WG часто
   «отлипает»), перепроверить.
3. **Здоров туннель выше по приоритету** → переключиться вверх (всегда стремимся
   к #1).
4. **Здоров туннель ниже по приоритету** → перейти на него.
5. **Ни один туннель не здоров, zapret доступен** → WAN-фолбэк, VPN-IP
   **вынимаются** из сета `nozapret`, чтобы zapret обходил их DPI на WAN
   (состояние `zapret`).
6. **Туннелей нет, zapret недоступен/падает** → обычный WAN (состояние `wan`).
   При **kill switch=1** вместо этого таблица 200 блэкхолится — трафик политики
   никогда не утекает (`killswitch`).

Демон всегда восстанавливается в сторону приоритета #1.

## Неразрушающее зондирование

Ключевая особенность: кандидаты проверяются **не ломая живой путь**. Кандидат
поднимается параллельно (`ifup`), и через него делается `ping -I <iface>`. Живой
туннель не разрывается, пока проверяются остальные.

Поскольку у эндпоинтов принудительно `route_allowed_ips=0` (чтобы поднятие
туннеля не перехватывало маршруты `main`), у них нет маршрута в `main` к публичным
целям. Поэтому проба ставит **изолированный** маршрут в отдельной таблице
`201` (приоритет правила 998) через source-IP кандидата только на время пинга —
ни `main`, ни таблица 200, ни пины DoH не затрагиваются.

## Проверки здоровья

- **Туннель:** `ping` целей `health_target` через интерфейс. Count=2 — первый
  пакет будит «спящий» туннель без keepalive (триггерит рукопожатие), второй
  подтверждает. Любой ответ = здоров.
- **WAN/zapret:** `curl health_url` через WAN решает, претендовать ли на ступень
  `zapret` или падать на `wan`. Ступень `zapret` берётся только если синк
  `nozapret` удался, `nfqws` реально работает и WAN отвечает.

## Kill switch

`killswitch=1`: при отсутствии здорового туннеля таблица 200 получает
`blackhole default` — отмеченный трафик политики **отбрасывается, а не утекает в
WAN**. Блэкхол ставится и заранее, на время разрушительного окна
рестарта/пробы (cold start или зависший текущий туннель), чтобы ничего не утекло.
`set_vpn` снимает блэкхол при восстановлении.

## Самовосстановление списков

- `ipsum` (и `ru` в режиме `full`) перезаливаются из nft-файла, если живой сет
  «просел» ниже минимума после перезагрузки fw4.
- При первом запуске демон в фоне дочитывает списки, которых ещё нет на диске
  (вместо ожидания ночного cron).
- `nozapret` перестраивается каждый раз при выборе VPN/деградации.

## Счётчик сбоев

`fail_count` (в `/var/run/wg-split-failcount`) растёт при каждом тике, когда ни
один туннель не здоров, и обнуляется при выборе VPN. Виден в hero-баннере LuCI и в
`wg-split-doctor`.

## Состояние

Активный путь хранится в `/var/run/wg-split-state`: `vpn:<iface>` | `zapret` |
`wan` | `killswitch`. Журнал переходов — `logread -e wg-split`.

См. [Диагностику](Diagnostics.md) для разбора, почему туннель считается мёртвым
(чаще всего — firewall, см. [Устранение неполадок](Troubleshooting.md)).
