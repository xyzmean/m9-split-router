# wg-split 2.0.0 Field Fixes (N1–N4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the four production-observed defects (N1–N4) found in the 27.06 live-router assessment, without touching the already-shipped 2.0.0 feature set.

**Architecture:** Two surgical shell changes. (a) `wg-split-sync-nozapret` gains an idempotency guard so it stops rebuilding the ~71k-entry nozapret set and logging on every 180 s failover tick when nothing changed — this single change fixes both N1 (log flood evicting syslog history) and N2 (redundant CPU churn). (b) `wg-split-doctor`'s list reporting stops showing `ru count:0` for a list that is in fact live (it feeds nozapret in blocklist/split mode) and exposes `file_count` alongside the live-set `count` so the two numbers are no longer conflated (N3 + N4). All new logic is extracted as pure POSIX-sh functions and unit-tested with bats, mirroring the existing test pattern.

**Tech Stack:** busybox/ash POSIX shell, nftables (`nft`), bats unit tests, shellcheck, GitHub Actions (`.github/workflows/test.yml`).

## Global Constraints

- **Shell dialect:** POSIX `sh` (busybox ash). `shellcheck -s sh` must pass — `test.yml` lints every `wg-split-*` script. No bashisms.
- **No `set -u`:** the scripts source `/lib/functions.sh`, which is not nounset-safe. Do not add `set -u`.
- **Pure functions only for tests:** unit-tested functions must be self-contained (`name()` at column 0, balanced braces, no `uci`/`nft`/`ip` calls) so `tests/helper.bash`'s `extract_fn` can lift them verbatim into CI.
- **Additive JSON only:** `wg-split-doctor --json` is consumed by the merged LuCI `status.js`. Only ADD JSON fields; never rename/remove existing ones (`name`, `enabled`, `count`, `min`, `file`, `age`, `ok`).
- **Logging helper:** use `log "..."` (defined in `common.sh` as `logger -t "$LOG_TAG"`). There is no `debug()` helper — do not invent one; suppress noise by NOT logging on no-op, not by adding a log level.
- **Self-heal must survive:** a fw4 reload can drain a live nft set; the idempotency guard must still rebuild when the live set is below its min floor.

- [ ] **Step 4: Add the pure guard function**

In `wg-split/files/usr/local/sbin/wg-split-sync-nozapret`, after the `. /usr/local/lib/wg-split/common.sh` line (currently line 14) and before the `[ "$ZAPRET_ENABLED" = "1" ]` guard, insert:

```sh
# The nozapret set is rebuilt on EVERY failover tick (set_vpn -> rebuild_nozapret).
# When inputs haven't changed and the live set is healthy, that rebuild + its log
# line is pure churn (N1 log-flood / N2 CPU). nozapret_current decides "already
# current?": same signature AND live set >= min floor AND not force-requested.
# Pure logic (no nft/uci) so it is unit-tested off-box via tests/helper.bash.
nozapret_current() {   # NEW_SIG OLD_SIG LIVE_COUNT MIN FORCED
    [ "$5" = 1 ] && return 1                         # forced -> rebuild
    [ -n "$1" ] && [ "$1" = "$2" ] || return 1       # sig empty/changed -> rebuild
    [ "${3:-0}" -ge "${4:-0}" ] 2>/dev/null || return 1  # live set drained -> rebuild
    return 0
}
```

Run: `bats tests/wg-split.bats -f nozapret_current`
Expected: PASS (5 tests).

- [ ] **Step 6: Wire the guard into the main flow**

In the same file, define the signature-file path next to the `TMP=` line (currently line 30). Change:

```sh
TMP="/tmp/wg-split-sync-nozapret.$$"
```

to:

```sh
TMP="/tmp/wg-split-sync-nozapret.$$"
NOZAPRET_SIG="/var/run/wg-split-nozapret.sig"
```

Then, between the min-count floor check (the `die "too few entries..."` block, currently ending line 49) and the `emit_nft_set_block` call (currently line 51), insert the skip-when-current guard:

```sh
# N1/N2: skip the rebuild + log entirely when the candidate set is byte-identical
# to the last synced one AND the live set is still healthy. INCLUDE_VPN is part of
# the signature so the degradation toggle (1->0) always forces a rebuild.
NEW_SIG="$INCLUDE_VPN:$(md5sum < "$TMP/all.lst" | cut -d' ' -f1)"
OLD_SIG="$(cat "$NOZAPRET_SIG" 2>/dev/null)"
LIVE="$(set_count "$ZAPRET_NOZAPRET_TABLE" "$ZAPRET_NOZAPRET_SET")"
if nozapret_current "$NEW_SIG" "$OLD_SIG" "${LIVE:-0}" "$ZAPRET_NOZAPRET_MIN" "${WG_SPLIT_FORCE_SYNC:-0}"; then
    exit 0
fi
```

Finally, replace the trailing log line (currently line 53–54):

```sh
nft -f "$TMP/sync.nft" || die "atomic nft sync failed (kept: $TMP/sync.nft)"
log "nozapret synced: $count entries"
```

with the version that records the new signature:

```sh
nft -f "$TMP/sync.nft" || die "atomic nft sync failed (kept: $TMP/sync.nft)"
printf '%s\n' "$NEW_SIG" > "$NOZAPRET_SIG"
log "nozapret synced: $count entries"
```
